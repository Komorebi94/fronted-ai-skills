# Self Review

## Before Finishing

Review the work for:

- The request is fully addressed without unrelated changes.
- Existing business logic and public behavior are preserved unless intentionally changed.
- Edge cases, invalid input, empty states, and error paths are considered.
- Types, imports, exports, and references are consistent.
- Naming, structure, and formatting match nearby code.
- Security and privacy risks are not introduced.
- Performance is reasonable for the likely scale and execution frequency.
- Tests or verification steps match the risk of the change.

Revise the work or response if the review finds a concrete issue.
